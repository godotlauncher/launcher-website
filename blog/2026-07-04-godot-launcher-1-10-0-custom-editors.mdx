Godot Launcher 1.10.0 makes custom-built Godot editor support a first-class part of the Installs and Projects workflows. You can register custom-built Godot editors next to official releases, select them for projects, and recover more gracefully when a project asks for an editor that is not installed.

<!-- truncate -->

**Highlights**

- Register custom-built Godot editors and select them for projects.
- Create a custom editor manifest from the Installs view when you do not already have one.
- Resolve missing editor requirements while importing projects.
- Preserve invalid VS Code config files as `.bad` copies and recreate the files the launcher needs.
- Rename launcher project entries from the project settings drawer.

## Register and use custom-built Godot editors

Registered custom-built Godot editors appear in **Installs** with a **Custom** badge. They can be selected for projects like regular installed editors.

If you do not already have a custom editor manifest, **Custom Editor -> Create custom editor manifest** is available from the Installs view. It writes `godotlauncher-editor-manifest.json` to the folder you choose and registers the editor after creation. The manifest format is defined by the [v1 engine manifest schema](https://github.com/godotlauncher/launcher/blob/main/schemas/v1/engine-manifest.json).

## Recover when a project asks for a missing editor

Adding an existing project is more forgiving in 1.10.0. If a project requests an editor that is not installed, Godot Launcher can show an **Editor version required** dialog instead of stopping immediately.

From there you can download the requested editor when available, use a compatible installed editor as a fallback, or add the project with the editor marked missing so you can resolve it later.

## Safer VS Code configuration recovery

Godot Launcher now handles invalid or unsupported VS Code config files more carefully. If `.vscode/settings.json`, `.vscode/extensions.json`, or `.vscode/launch.json` cannot be safely read, the launcher preserves the original file as a timestamped `.bad` copy, recreates the config it needs, and warns you with the recovered paths.

## More project workflow improvements

This release also adds project rename settings, richer project rows with icon metadata when available, more predictable recent-project ordering, and clearer manual update messaging on rpm-ostree systems.

## Update to 1.10.0

Grab the latest installer from the download page when 1.10.0 is available. If you already use the launcher updater on a supported updater path, the in-app update flow can guide you through the update.
